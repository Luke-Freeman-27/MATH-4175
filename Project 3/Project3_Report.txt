2. Linear approximation trail (sketch in separate PDF)
----------------------------------------------------------------------

Cipher: three rounds + final XOR with K4 (Figure 1). After each S-layer the
6-bit permutation sends
  D1=B1, D2=B3, D3=B5, D4=B2, D5=B4, D6=B6
(same for F‚ÜíG).

Plaintext mask for P1‚äïP2‚äïP4‚äïP5 (with P1 the LEFTmost handout bit):
  (1,1,0,1,1,0) on (A1‚Ä¶A6) after XOR with K1.

Round 1 ‚Äî both active S-boxes have input mask a=6 (binary 110). In the LAT,
row 6 has a single nonzero column b=4 with NL‚àí4 = ‚àí4, Œµ(6,4) = ‚àí1/2.
So both S11 and S12 use output mask 4. Output mask on B is (1,0,0,1,0,0).

Round 2 ‚Äî permute B to E = (1,0,0,0,1,0). Split masks a=4 on S21 and a=2 on S22.
One useful choice with large |bias| is (b,b')=(2,2): NL(4,2)=+2 and NL(2,2)=‚àí2,
so Œµ = 1/4 and Œµ = ‚àí1/4; piling-up inside the round gives
  Œµ_round2 = 2 ¬∑ (1/4) ¬∑ (‚àí1/4) = ‚àí1/8.

Piling-up across the two active rounds gives
  Œµ_total = 2 ¬∑ Œµ_round1 ¬∑ Œµ_round2
with Œµ_round1 = 2¬∑(‚àí1/2)¬∑(‚àí1/2) = 1/2, hence
  Œµ_total = 2 ¬∑ (1/2) ¬∑ (‚àí1/8) = ‚àí1/8.

The guess farthest from the random baseline 3/6 is 000 with 6/6 zeros, so the

----------------------------------------------------------------------
6. Why the second bit is not determined by this statistic (conceptually)
----------------------------------------------------------------------

The counter uses one linear relation: P1‚äïP2‚äïP4‚äïP5‚äïH1.  
After partial decryption, H1 is a nonlinear function of (C1,C2,C3) and the three guessed
subkey bits. This linear test gives a strong bias for the first/third guessed bits, but it
does not isolate the middle guessed bit by itself.  
So this specific statistic is not enough to uniquely determine K4's second bit; a different
approximation (or more data) is needed to resolve it.
